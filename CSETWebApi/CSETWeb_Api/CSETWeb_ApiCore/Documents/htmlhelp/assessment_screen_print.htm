Assessment Screen

 
The assessment screen includes a series of questions, requirements, or practices for which assessors provide responses. Responses provided on the assessment screen are compiled and presented in the Results section.
 
The selected assessment type, the assessment mode, the security assurance level (SAL), the components selected within a network diagram, and the information provided on the Prepare pages may impact the assessment structure and which questions/requirements/practices display on the assessment screen. These settings and selections help customize the assessment to better fit the organization, business unit, or critical service being assessed.
 
See the tutorial page (if available) in CSET for information specific to the assessment type.
 
If completing an assessment that contains a Network Diagram, see Diagram Component Questions for more information.
 

Assessment Modes

  • Use the assessment mode toggle to view the assessment screen in one of the following modes:
  • Questions Mode
  • In this mode, standard requirements are reworked into simple and straightforward questions.
  • Note: In some cases, a single requirement may be reworked and split into multiple questions.
  • The related requirement display in the question's Details/Standards section.
  • Requirements Mode
  • This mode displays the exact requirement text and uses the practice identifier from standard.
  • The related question(s) display in the requirement's Details/Standards section.
  • Note: Some assessment types do not have multiple assessment modes, therefore the toggle does not always display.

Number of Answered Questions/Requirements

  • In Questions Mode:
  • The counter indicates the number of answered questions out of the total number displayed on the assessment screen.
  • In Requirements Mode:
  • The counter indicates the number of answered requirements out of the total number displayed on the assessment screen.

Collapse All/Expand All/Filter

  • Select the Collapse All button to close all sub-domains or categories and hide all questions, requirements, or practices.
  • Select the Expand All button to open all sub-domains or categories and show all questions, requirements, or practices.
  • Select the Filter button to open the Question Filters window.
  • Users can filter questions/requirements in two ways:
  • Enter a keyword into the Search bar to filter and search for questions/requirements.
  • A filtered list of questions/requirements will display on the assessment screen based on the entered keyword.
  • Select or deselect the checkboxes to show/hide certain questions/requirements.
  • A filtered list of questions/requirements will display on the assessment screen based on the selected criteria.

Auto-load Guidance

  • Check the Auto-load Guidance checkbox to automatically display the Supplemental Guidance section within the Question/Requirement buttons.

Assessment Header

  • The assessment header displays the title of the assessment screen.

Domain/Main Category

  • Domains/main categories are high level groupings that each contain a set of associated questions/requirements.

Sub-Domain/Sub-Category

  • Sub-domains/categories represent the desired cybersecurity outcome or mitigation that each of its associated questions/requirements aims to evaluate.
  • For standard-based assessments, it's typically the category specified in the question's/requirement's corresponding standard.
  • Some assessments include a brief overview of its purpose and suggested approaches alongside each sub-domain.
  • Based on the assessment type and assessment mode, the sub-domain/sub-category may contain its own question.

Progress Indicator

  • The progress indicator shows the user's progress in answering all questions/requirements within the sub-domain or category. The progress indicator will display a checkmark once all questions/requirements have been answered.

Practice Identifier

  • Each question/requirement has an assigned practice identifier.
  • Based on the assessment type and assessment mode, some can be question numbers or a combination of unique characters to indicate characteristics such as the domain, the level and/or requirement number.
  • For standard-based assessments, it's typically the identifier specified in the question's/requirement's corresponding standard.

Question/Requirement

  • Each question/requirement is designed to evaluate if the organization has implemented the associated sub-domains.
  • They are typically mitigation(s) that organizations should implement to achieve the corresponding outcome and reduce cybersecurity risks.
  • For standard-based assessments, questions are developed from the associated requirements, whereas requirements are the exact requirement text from the corresponding standard.

Answer Buttons

  • Select the applicable answer for each question.
  • Answer buttons vary based on the selected assessment type and assessment mode. Common answer buttons and their general definition are listed below.
  • Yes: The organization fully performs the activity specified in the question.
  • Incomplete: The organization partially performs the activity.
  • No: The organization does not perform the activity at all.
  • N/A: The practice or requirement does not apply to the organization.
  • Note: Any question marked as "N/A" will not show up in the online analysis or reports as a deficiency or missed answer; nor will they count as a "Yes" answer.
  • Alt: The organization has implemented an alternative approach or compensating measures to meet the requirement.
  • When selected, a text box displays for the user to provide a description/justification of the alternative measure.
  • Note: An alternate method is scored in a positive way, similar to a "Yes "answer.
  • Implemented: An organization has implemented and continues to maintain the recommended actions, or a suitable alternative, necessary to achieve the stated outcome.
  • In Progress: An organization is in the process of implementing the recommended actions within a goal, or a suitable alternative, to achieve the stated outcome.
  • Scoped: An organization has identified the full set of activities required to meet the stated outcome of a goal.
  • Not Implemented: An organization has no immediate plans to implement the recommended actions for a goal.
  • Unanswered questions are typically calculated as a “No” or "Not" response.

Mark for Review

  • Select the Mark for Review button to flag a question/requirement for future review.
  • These questions/requirements are compiled into the Comments and Marked for Review report.

Question/Requirement Buttons

  • Each question/requirement includes a set of buttons that display the following sections:
  • Details/Standard (if applicable)
  • Supplemental Guidance
  • Comments
  • Artifacts/Documents
  • References
  • Observations
  • Feedback

Reviewed

  • Select the Reviewed button to mark the question/requirement as reviewed.
  • Note: The Reviewed button does not display for all assessment types.
